Styling components

Using CSS custom properties

Studiö components can be styled using CSS custom properties. This works for the most common use cases and is the best way to use the components as black boxes.
For instance, the --studio-surface-color custom property holds the background color to use for a --studio-led.

Using parts

As an alternative to the CSS custom properties, Studiö components can be styled using part pseudo-elements (shadow parts). For reasons explained below, we recommend the use of parts only when no CSS custom property will provide what you need.
Using CSS, each part can be styled at the element selector, class selector or id selector level. The use of parts offers more flexibility than using the CSS custom properties method, however parts do not treat the components as black boxes, meaning that future changes to a component's implementation may break the styling that you apply to parts.
Styling with parts offers the most flexibility because you can style more than what is possible strictly with the provided CSS custom properties. However, be aware that your styling of the parts may break the intended presentation of a component, for instance you may use position: absolute but that could structurally alter how a component is represented or behaves.
Animations currently do now work on parts because of scoping of the @keyframes. See here.
The documentation for each component describes which parts are available. There is a styling pattern shared across components, where applicable.
